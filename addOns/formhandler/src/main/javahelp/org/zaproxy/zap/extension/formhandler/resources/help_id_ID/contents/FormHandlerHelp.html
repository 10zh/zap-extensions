Bentuk Handler

Ekstensi bentuk handler ini memungkinkan untuk konfigurasi khusus untuk nilai yang digunakan dalam bentuk yang berdasarkan nama bidang.

Deskripsi

With this add-on the user can create new fields for fields that they wish to set the value of. Additionally, the user can modify any existing field's name and value to fit the fields that they are trying to set. Bidang apapun juga dapat dinonaktifkan atau dihapus oleh pengguna sehingga nilai-nilai adat tidak digunakan. Field names, values, enabled, and regex status will be stored even after a ZAP session has concluded.

Add-on ini menyediakan satu set default nama bidang yang dapat ditemukan pada formulir yang sesuai nilai default untuk masing-masing bidang. Nilai default ini dapat dinonaktifkan, diubah dan dihapus jika keinginan pengguna.

Menggunakan Form Handler

Bentuk Handler dapat ditemukan dalam Pilihan. Memilih tab Bentuk Handler akan menampilkan tabel dengan semua nilai yang ada yang saat ini didefinisikan. Contoh ditunjukkan di bawah ini.

Menambahkan Bidang Baru

Untuk menentukan nilai bidang tambahan Tambahkan tombol yang dapat diklik. Ini akan memunculkan dialog bagi pengguna untuk memberikan bidang informasi.

Harap perhatikan berikut ini saat menambahkan bidang baru:

  • A field name or regex pattern must be provided.
  • The field name cannot already exist in the table or a warning will occur.
  • Nama bidang yang disediakan untuk Bentuk Handler harus sesuai dengan bidang ID pada bentuk yang anda inginkan untuk mendefinisikan.
  • The field name is not case sensitive, unless using regex.
  • Simple name matches will be attempted before regex matches.
  • A value does not have to be provided. This will result in an empty string being used.
  • The value field is case sensitive, and will be seen in the POST.

Memodifikasi Bidang yang ada

Ekstensi ini memungkinkan pengguna untuk mengubah nilai dan nama bidang yang ada. Hal ini dapat dilakukan dengan memilih bidang yang ingin anda ubah dan klik pada Ubah tombol.

Saat memodifikasi bidang pengguna dibatasi oleh aturan yang sama seperti membuat yang baru. Kendala yang paling umum ketika memodifikasi sebuah bidang adalah nama yang tidak bisa sama dengan bidang yang ada. (yaitu bukan bidang duplikat)

Menghapus Bidang

Pengguna dapat menghapus bidang kapan saja. Untuk melakukannya cukup pilih bidang yang ingin anda hapus dan klik pada tombol Hapus. Harap dicatat bahwa anda akan diminta untuk mengkonfirmasi penghapusan sebelum melanjutkan kecuali kotak Hapus tanpa Konfirmasi dicentang.

Bentuk Bidang Yang Tidak Didefinisikan

Pengguna mungkin menemukan bahwa mereka tidak ingin untuk mendefinisikan setiap bidang dalam suatu bentuk, tetapi lebih menentukan beberapa pilihan dalam bentuk.

Setiap bidang yang tidak didefinisikan dalam bentuk handler add-on ini akan menggunakan nilai default ZAP yang dihasilkan untuk menyesuaikan pertandingan masing-masing jenis bidang.(Misalnya: bidang teks akan memiliki nilai "ZAP")

Statistics

A number of statistics are exposed via the add-on for add, modify, and remove operations.
In the following examples {name} will be the actual name of the Form Handler field.

  • stats.formhandler.add
  • stats.formhandler.add.{name}
  • stats.formhandler.modify
  • stats.formhandler.modify.{name}
  • stats.formhandler.remove
  • stats.formhandler.remove.{name}
