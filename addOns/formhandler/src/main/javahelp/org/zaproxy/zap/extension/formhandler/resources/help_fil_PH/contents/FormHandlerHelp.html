Tagapamahala ng Form

Ang ekstensyong sa Tagapamahala ng Form ay magpapahintulot para sa mga kustom na kompigurasyon ng mga balyu na ginagamit sa mga form base sa mga pangalan ng field.

Deskripsyon

With this add-on the user can create new fields for fields that they wish to set the value of. Additionally, the user can modify any existing field's name and value to fit the fields that they are trying to set. Ang kahit anong mga field ay maaring hindi mapagana o maalis ng tagagamit upang ang mga kustom na balyu ay hindi magamit. Field names, values, enabled, and regex status will be stored even after a ZAP session has concluded.

Ang add-on na ito ay nagbibigay ng mga pangkat na default na mga pangalan ng field na maaring mahanap sa isang form at ng kaukulan na balyu para sa bawat default na field. Ang mga default na balyung ito ay maaring hindi mapagana, mabago at mabura kung ang tagagamit ay maghahangad nito.

Paggamit sa Tagapamahala ng Form

Ang Tagapalamahal ng Form ay maaring makita sa Mga Pagpipilian. Ang pagpili ng tab sa Tagapamahala ng Form ay magpapakita ng isang talaan na may lahat ng umiiral na mga balyu na kasalukuyang binigyan ng kahulugan. Ang halimbawa ay makikita sa ibabaw.

Pagdagdag Ng Isang Bagong Field

Upang mabigyang ng kahulugan ang karagdagang mga balyu ng field ang Magdagdag na button ay maaring mapindot. Ito ay magdadala palabas ng isang dialogo para sa manggagamit na magbigay ng impormasyon sa field.

Mangyari lamang na tandaan ang mga sumusunod kapag magdadagdag ng isang bagong field:

  • A field name or regex pattern must be provided.
  • The field name cannot already exist in the table or a warning will occur.
  • Ang pangalan ng field na inilagay sa Tagapamahala ng Form ay kinakailangan magtugma sa ID ng field sa form na gusto mong bigyan ng kahulugan.
  • The field name is not case sensitive, unless using regex.
  • Simple name matches will be attempted before regex matches.
  • A value does not have to be provided. This will result in an empty string being used.
  • The value field is case sensitive, and will be seen in the POST.

Pagbabago Ng Umiiral Na Field

Ang ekstensyon ay magpapahintulot sa tagagamit na baguhin ang mga balyu at pangalan ng umiiral na mga field. Ito ay maaring magawa sa pamamagitan ng pagpili ng field na gusto mong baguhin at pagpindot sa Baguhin na button.

Kapag nagbabago ng field ang tagagamit ay mapipilit sa parehong mga patakaran sa paggawa ng isang bago. Ang pinaka karaniwang pagpipilit kapag nagbabago ng field ay ang pangalan na hindi maaring maging pareho sa isang umiiral na field. (i.e Walang dobleng mga field)

Pag-aalis ng mga Field

Ang taggagamit ay maaring mag-alis ng mga field sa kahit anong oras. Upang magawa ito simpleng pumili ng field na gusto mong itanggal at pindutin sa Tanggalin na button. Mangyari lamang na tandaan na ikaw ay mauudyok na ikompirma ang pag-alis bago magpatuloy hanggang sa ang Pagtanggal ng walang Kompirmasyon na kahon ay naka-tsek.

Mga Field Ng Form Na Hindi Binigyan Ng Kahulugan

Ang taggagamit ay maaring makatanto na hindi nila gustong magbigay ng kahulugan sa kada field sa loob ng isang form, sa halip ay magbibigay kahulugan sa iilang mga napili sa loob ng form.

Ang kahit anong field na hindi nabigyan ng kahulugan sa loob ng tagapamahala ng form na add-on ay gagamit ng default na balyu ng ZAP na nalikha upang matugma ang bawat uri ng field.(Halimbawa: Ang tekstong field ay magkakaroon ng balyu "ZAP")

Statistics

A number of statistics are exposed via the add-on for add, modify, and remove operations.
In the following examples {name} will be the actual name of the Form Handler field.

  • stats.formhandler.add
  • stats.formhandler.add.{name}
  • stats.formhandler.modify
  • stats.formhandler.modify.{name}
  • stats.formhandler.remove
  • stats.formhandler.remove.{name}
