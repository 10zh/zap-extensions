Платформа автоматизации - Окружающая среда

Этот раздел файла конфигурации YAML определяет приложения, с которыми могут работать остальные задания.

Платформа автоматизации поддерживает подмножество механизмов аутентификации , поддерживаемых ZAP. 

env:                                   # Среда, обязательно 
  contexts :                           # Список из 1 или более контекстов, обязательно 
    - name: context 1                  # Имя, которое будет использоваться для ссылки на этот контекст в других вакансиях, обязательно 
      urls:                            # Обязательный список URL-адресов верхнего уровня, все, что находится под каждым URL-адресом, будет включено 
      includePaths:                    # Необязательный список включаемых регулярных выражений 
      excludePaths:                    # Необязательный список исключенных регулярных выражений 
      authentication:
        method:                        # Строка, одна из 'manual', 'http' - со временем будет охватывать все методы, поддерживаемые ZAP 
        parameters:
          hostname:                    # Строка, только для метода http 
          port:                        # Int, только для метода http 
          realm:                       # Строка, только для метода http 
        verification:
          method:                      # Строка, одно из «ответ», «запрос», «оба», «опрос» 
          loggedInRegex:               # Строка, шаблон регулярного выражения для определения входа в систему 
          loggedOutRegex:              # Строка, шаблон регулярного выражения для определения выхода из системы 
          pollFrequency:               # Int, частота опроса, только для проверки 'опроса' 
          pollUnits:                   # Строка, единицы опроса, один из «запросов», «секунд», только для проверки «опроса» 
          pollUrl:                     # Строка, URL для опроса, только для проверки 'опроса' 
pollPostData:                # Строка, данные публикации для включения в пул, только для проверки "опроса" 
          pollAdditionalHeaders:       # Список дополнительных заголовков для запроса на опрос, только для проверки 'опроса' 
          - header:                    # Название заголовка 
            value:                     # Значение заголовка 
      sessionManagement:
        method:                        # Строка, одно из 'cookie', 'http', 'script' 
        script:                        # Строка, путь к скрипту, только для управления сеансом «скрипт» 
        scriptEngine:                  # Строка, имя используемого скриптового движка, только для управления сеансом «скрипт». 
      users:                           # Список одного или нескольких пользователей для аутентификации 
      - name:                          # Строка, имя, которое будет использоваться заданиями 
        username:                      # Строка, имя пользователя для использования при аутентификации, поддерживаются vars ( переменные среды)
        password:                      # Строка, пароль для использования при аутентификации, поддерживаются переменные 
  vars:                                # Список из 0 или более пользовательских переменных, которые будут использоваться в файле конфигурации 
    myVarOne: CustomConfigVarOne       # Может использоваться как $ {myVarOne} в любом месте конфигурации 
    myVarTwo: ${myVarOne}.VarTwo       # Может ссылаться на другие переменные ( среды)   
  parameters:
    failOnError: true                  # Если установлен выход при ошибке       
    failOnWarning: false               # Если установлен выход по предупреждению 
    progressToStdout: true             # Если установлено, прогресс выполнения задания будет записываться в стандартный вывод.
Переменные системной среды также можно использовать в конфигурации таким же образом, как указано выше. Если есть две переменные с тем же именем предпочтительнее использовать значение системной переменной.
