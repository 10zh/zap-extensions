Mga Patakaran ng Pasibong Pagsusuri-Beta

Ang sumusunod na mga patakaran ng kalidad ng pasibong pagsusuri ng beta ay kasama sa add-on:

Hindi tugmang Charset

Ang pagsiyasat ng mga tinukoy sa mga pagtugon kung saan ang nilalaman ng HTTP-Tipo ng header ng mga idiniklara ay ang pagkakaiba ng charset mula sa tinukoy na charset sa pamamagitan ng HTML o XML. Kung mayroong isang hindi tugmang charset sa pagitan ng HTTP header at katawan ng nilalaman ng mga Web Browser ay maaaring ipilit sa isang hindi kanais-nais na nilalaman-sniffing mode upang matukoy ang nilalaman ng tama ang karakter na itinakda.

Ang nagsusuri ay mga naghahawak ng mga sari-saring mga kondisyon depende sa itinakda ng Threshold para sa nagsusuri, tulad ng mga sumusunod:

  • Lahat ng mga Threshold:
    • Header Laban sa Nilalaman ng Meta-Tipo ng Charset - Ang deklarasyon ng Nilalaman ng HTTP-Tipo ng header ay hindi tugma kung ano ang idineklara sa isang Nilalaman ng Meta-Tipo ng tag.
    • Header Laban ng Meta Charset - Ang deklarasyon ng Nilalaman ng HTTP-Tipo ng header ay hindi tugma sa kung ano ang idineklara sa isang Meta Charset ng tag.
  • Mababang Threshold:
    • Nilalaman ng Meta-Tipo ng Nawawalang Charset - Ang pag-tugon ay hindi naglalaman ng isang nilalaman ng META-Tipo ng deklarasyon, na maaaring makita ng lumang mga kliyente.
    • Meta Charset Laban sa Nilalaman ng Meta-Tipo ng Charset - Ang mga nilalaman ng parehong pagtugon ng deklarasyon na Nilalaman ng Meta-Tipo at ang deklarasyon ng Meta Charset, at sila'y hindi magkatugma.
Ang mga karagdagang saggunian:

Cookie - Maluwag na saklaw

Ang mga Cookie ay maaaring i-saklaw ng dominyon o daan. Itong pagsisiyasat ay nag-aalala lamang kasama sa saklaw ng dominyon. Ang saklaw ng dominyon ay inilapat sa isang cookie na mga tumutukoy kung aling mga dominyon ang maa-access ito. Halimbawa, ang isang cookie ay maaaring strikto sa pagsaklaw sa isang subdomain hal. www.nottrusted.com, o luwagan ang sinaklawan na parent dominyon hal. nottrusted.com. Sa huling kaso, anuman ang subdomain ay nottrusted.com ay maaaring i-access ang cookie. Luwagan ang saklaw ng mga cookie na karaniwan sa mga mega-aplikasyon tulad ng google.com at live.com.

CSRF Mga Countermeasure

Ang mga CSRFCountemeasure na plugin ay kinikilala ang "potensyal" na mga kahinaan kasama ang kakulangan ng kilalang mga CSRF countermeasure ng mga pahina kasama ang mga form.
I-post ang 2.5.0 na maaaring matukoy ng isang kuwit na paghiwalayin ang listahan ng mga pagkakakilanlan sa mga rules.csrf.ignorelist parametro sa pamamagitan ng mga pagpipilian 'Kumpigura ng patakaran' ng panel. Ang anumang mga FORM kasama ng isang pangalan o ID ng mga tumutugma sa isa sa mga tumutukoy ay maaaring hindi pansinin habang sinusuri para sa nawawalang mga Anti-CSRF. Gumamit lamang ng mga tampok na hindi pinapansin ang mga FORM na alam mong ito ay ligtas, halimbawa hanapin ang mga form.

Paglalantad ng Impormasyon: Mga mali sa pag-debug

Itong Pasibong pagsusuri na mga nagsisiyasat sa nilalaman ng mga pagtugon ng web para sa kilalang kamalian ng Debug sa mensahe sa mga fragments. Ang pag-access sa mga naturang mga detalye ay maaaring magbigay ng isang malisosyong indibidwal kasama na nangangahulugan na kung saan ay naglalayon na abusuhin ang web site. Maaari din mag-leak ang datos nila ng partikular na hindi sinasadya para sa pagkonsumo ng pagtatapos ng gumagamit.

Paglalantad ng Impormasyon: Sa URL

Ang mga pagtatangka na tukuyin ang pag-iral ng mga detalyeng sensitibo sa loob ng pagbisita sa mga URl ng mga ito. (maaari nitong isama sa mga parametro, mga pangalan ng dokumento, mga pangalan ng taga-pangasiwa, at iba pa).

Paglalantad ng Impormasyon: Ang Tumutukoy

Kinikilala nito ang pag-iral ng mga detalyeng sensitibo sa loob ng header ng taga-tukoy sa field ng mga kahilingan ng HTTP (maaari nitong isama ang mga parametro, mga pangalan ng dokumento, mga pangalan ng taga-pangasiwa at iba pa).

Paglalantad ng Impormation: Mga Komentong Kahina-hinala

Sinusuri nito ang nilalaman ng web upang tukuyin ang mga komento kung saan naglalaman ng may potensyal na sensitibong mga detalye. Na kung saan ay naglalayong umatake o malantad ang hindi intensyon na datos.

Hindi ligtas na nagpapatunay

Ang HTTP basic o ang pagpapatunay ng digest ay ginamit sa isang hindi ligtas na koneksyon. Ang kredensyal ay maaaring basahin at pagkatapos gamitin muli ng isang taong may access sa network.

Hindi ligtas ang JSF ViewState

Ang pag tugon sa mga sumusunod na URL ay naglalaman ng isang mahalaga ng ViewState na walang mga proteksyon.

Servlet ng Parametrong Polusyon

Sa paghahanap ng pagtugon para sa mga form ng HTML kung saan ay bigo na matukoy ang isang aksyon ng elemento. Beryon na 3 ng Java Servlet spec ang tawag para sa pagsasama-sama ng mga katanungan ng string at ang datos ng post na mga elemento kung saan ay maaaring may resulta na hindi sinasadyang paghawak ng gumagamit para makontrol ang datos. Ito ay maaaring maging epekto sa ibang mga framework at mga teknolohiya din.

VIEWSTATE

Pagtatangka sa pagtukoy ng mga parametro ng VIEWSTATE at analisahin ang nasabing mga parametro para sa iba't ibang mga pinakamahuhusay na pagsasanay o proteksyon ng mga sukat tulad ng:
  • Ang pagbatay sa ASP.NET 1.0 at 1.1.
  • Ang kakulangan ng lagda ng VIEWSTATE.
  • Paghatiin ang VIEWSTATE.
  • Naglalaman ng email at mga IP pattern ang VIEWSTATE.
