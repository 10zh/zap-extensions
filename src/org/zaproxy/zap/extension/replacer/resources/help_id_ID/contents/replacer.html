Menggantikan

Cara mudah untuk mengganti string dalam permintaan dan tanggapan.

Hal ini dapat diakses melalui Opsi dan secara default ini dapat diakses secara cepat melalui '<ctrl> R' hotkey.

The Replacer Options panel allows you to define as many replacement rules as you need.
Each rule is defined by the following fields:

Description

A description that allows you to identify the rule.

Match Type

May be one of:

Request Header (will add if not present)

In this case the 'Match String' will be treated as a header name. Jika header hadir maka nilainya akan diganti dengan teks pengganti. Jika penggantian teks kosong maka header akan dihapus (jika ada). Jika header tidak hadir dan penggantian teks tidak kosong maka header akan ditambahkan.

Request Header String

In this case the 'Match String' will be treated as a string or regex expression. Jika hadir di header permintaan maka akan diganti dengan teks pengganti.

Request Body String

In this case the 'Match String' will be treated as a string or regex expression. Jika hal ini hadir dalam permintaan tubuh maka akan diganti dengan teks pengganti.

Response Header (will add if not present)

In this case the 'Match String' will be treated as a header name. Jika header hadir maka nilainya akan diganti dengan teks pengganti. Jika penggantian teks kosong maka header akan dihapus (jika ada). Jika header tidak hadir dan penggantian teks tidak kosong maka header akan ditambahkan.

Response Header String

In this case the 'Match String' will be treated as a string or regex expression. Jika hadir di respon header maka akan diganti dengan teks pengganti.

Response Body String

In this case the 'Match String' will be treated as a string or regex expression. Jika hal ini hadir dalam respon tubuh maka akan diganti dengan teks pengganti.

Match String

The string that will be used to identify what should be replaced - see Match Type for details.

Match Regex

If set then the Match String will be treated as a regex expression. Opsi ini dinonaktifkan ketika pencocokan header.

Replacement String

The new string that will replace the specified selection.

Enable

If not set then the rule will not apply.

Advanced

If set then an additional tab will be show which allows you to specify which 'initiators' the rule should apply to. Each ZAP component which can send and receive messages is a separate initiator, so this gives you a very fine grain control over exactly when the rule should apply.

If you need even more control over when and how text should be replaced then you can use the ZAP Script Console which gives you complete control over the ZAP messages.
