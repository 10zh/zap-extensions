AJAX laba-Laba dialog

Dialog ini meluncurkan AJAX laba-Laba.

Cakupan

Tab pertama memungkinkan anda untuk mengubah fitur utama seperti:

Titik awal

URL yang AJAX laba-laba akan mulai merangkak dari, atau konteks (dalam hal ini akan digunakan salah satu Url yang berada dalam konteks sebagai titik awal).

Konteks

Memungkinkan untuk memilih Konteks spidered.

Pengguna

Memungkinkan untuk memilih salah satu pengguna yang tersedia dari yang dipilih konteks, untuk melakukan spider scan sebagai pengguna (ZAP akan (kembali)mengotentikasi sebagai pengguna bila diperlukan).

Just in scope

Jika ditetapkan maka setiap Url yang di luar dari ruang lingkup yang akan diabaikan.

Note: The option Just in scope is mutually exclusive with Context option, if one is used the other is ignored.

Spider Subtree Only

Jika diatur kemudian laba-laba hanya akan mengakses sumber daya yang berada di bawah titik awal (URI). Ketika mengevaluasi jika sumber daya ini ditemukan dalam waktu yang telah ditentukan sub, laba-laba hanya mempertimbangkan skema, host, port, dan path komponen dari URI.

Browser

The type of browser to use.
Browser hanya akan ditampilkan jika ZAP telah cukup konfigurasi untuk menjalankan mereka.
Mereka mungkin masih gagal untuk dijalankan jika browser tidak dapat ditemukan atau jika informasi konfigurasi yang salah.

Show advanced options

Jika dipilih maka tab Options akan ditampilkan.

Options

Tab ini memungkinkan anda untuk mengubah pilihan, termasuk:

Number of browser windows to open

You can configure the number of windows to be used by AJAX Spider.
The more windows, the faster the process will be.

Maximum crawl depth

Kedalaman maksimum crawler dapat mencapai. Nol berarti kedalaman tak terbatas.

Maximum crawl states

Jumlah maksimum menyatakan bahwa crawler harus merangkak. Nol berarti tidak terbatas merangkak serikat.

Maximum duration

Waktu maksimum yang crawler yang diperbolehkan untuk berjalan. Nol berarti tak terbatas waktu berjalan.

Event wait time

The time to wait after a client side event is fired.

Reload wait time

The time to wait after URL is loaded.
