Active Scan Rules - alpha

The following alpha quality active scan rules are included in this add-on:

Apache Range Header DoS (CVE-2011-3192)

Tests to see if the server is vulnerable to the Apache Range Header Denial of Service issue, by requesting eleven (11) different byte ranges. Sebelas rentang adalah salah satu lebih dari diterima oleh server ditambal/tetap.

Cookie Slack Detector

Tests cookies to detect if some have no effect on response size when omitted, especially cookies containing the name "session" or "userid"

ELMAH Scanner

Tests to see if the Error Logging Modules and Handlers (elmah.axd) HTTP Module is available. Walaupun modul ini sangat berguna untuk para pengembang dan pemangku kepentingan lainnya ini juga dapat bocor sejumlah besar informasi yang seorang analis keamanan atau individu berbahaya mungkin akan tertarik dalam.

Pemindai ELMAH berbasis Microsoft Technology: IIS, Windows, ASP, dan MSSQL.

Example File Active Scanner

This implements an example active scan rule that loads strings from a file that the user can edit.
For more details see: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Example Simple Active Scanner

This implements a very simple example active scan rule.
For more details see: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Expression Language Injection

The software constructs all or part of an expression language (EL) statement in a Java Server Page (JSP) using externally-influenced input from an upstream component, but it does not neutralize or incorrectly neutralizes special elements that could modify the intended EL statement before it is executed. Dalam versi tertentu dari Spring 3.0.5 dan sebelumnya, ada kerentanan (CVE-2011-2730) dalam bahasa ekspresi akan dievaluasi dua kali, yang secara efektif terkena aplikasi untuk EL injeksi. Namun, bahkan untuk versi yang lebih baru, kelemahan ini masih mungkin tergantung pada konfigurasi.

GET for POST Scanner

This scanner takes application/x-www-form-urlencoded POST requests, changes the parameters from POST to GET and resubmits the request. Jika MENDAPATKAN respon yang sama seperti aslinya POSTING respon maka alarm akan berbunyi. Sementara ini tidak selalu mewakili kelemahan keamanan kepada dirinya sendiri mungkin mengindikasikan bahwa serangan atau kelemahan lain dapat diperluas atau disederhanakan. (Such as a POST based Cross-Site Scripting (XSS) attack being changed to GET.)

HTTP Only Site

This active scanner checks whether an HTTP site is served under HTTPS.

Httpoxy

Pemindai aktif ini memeriksa apakah sebuah situs menggunakan header HTTP Proxy yang ditentukan dalam permintaan.
Ini membuat proxy HTTP yang mendengarkan semua antarmuka pada port bebas yang ditetapkan secara acak. Kemudian mengirimkan serangkaian permintaan ke server target dengan header HTTP Proxy yang disetel ke masing-masing yang tersedia Alamat IP dan port yang didengarkannya. Jika permintaan diterima di port baru maka server sangat mungkin rentan.
PENTING - komputer yang dijalankan ZAP harus menerima permintaan masuk pada port yang sewenang-wenang - Jika firewall mencegah koneksi masuk maka aturan ini tidak akan berhasil.

HTTPS sebagai pemindai HTTP

Pemindai aktif ini mencoba mengakses konten yang awalnya diakses melalui HTTPS (SSL/TLS) melalui HTTP.

Injeksi LDAP

Injeksi LDAP dimungkinkan. Mungkin bagi penyerang untuk memotong kontrol otentikasi, dan untuk melihat dan memodifikasi data sewenang-wenang dalam direktori LDAP.

SQL Injection - MsSQL

This active scanner attempts to inject MsSQL specific sleep commands into parameter values and analyzes the server's response time to see if the sleep is effectively executed on the server (indicating a successful SQL injection attack).

Pengungkapan sumber kode - Inklusi Berkas

Menggunakan teknik penyisipan berkas lokal untuk memindai berkas yang berisi kode sumber pada server web.

Pengungkapan Kode Sumber - Git

Menggunakan teknik repositori kode sumber Git untuk memindai file yang berisi kode sumber pada server web.

Pemindai jejak.axd

Pengujian untuk melihat apakah jejak penonton (jejak.axd) tersedia. Meskipun komponen ini mudah bagi pengembang dan pemangku kepentingan lainnya dapat membocorkan sejumlah besar informasi yang analis keamanan atau individu jahat mungkin tertarik.

Pemindai jejak.axd menargetkan teknologi berbasis microsoft: IIS, windows, ASP, dan MSSQL.

User Agent Fuzzer

This active scanner checks for differences in response based on fuzzed User Agent (eg. mobile sites, access as a Search Engine Crawler). Pemindai membandingkan kode status tanggapan dan kode campuran dari tanggapan tubu dengan tanggapan asli.
