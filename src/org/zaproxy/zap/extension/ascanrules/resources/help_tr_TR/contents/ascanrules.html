Active Scan Rules

Bu eklentide bulunan aktif tarama kuralları şunlardır:

Buffer Overflow

Derlenmiş kodda buffer overflow'ların göstergelerini arar. Bunu büyük bir yazı girdisi salarak ve kod çakışmalarına ve anormal oturum kapanışlarına bakarak yapar.

Kod Enjeksiyonu

Bu kural PHP ve ASP saldırı string'lerini URL parametreleri için değerler olarak bir talepte bildirir ve o değerlerin sunucu tarafından değerlendirilip değerlendirilmediğini görmek için yanıtı inceler. İlk olarak, bir token yazdırmak için enjekte edilmiş PHP talimatları içeren talepler oluşturulur ve gönderilir. Token ve yanıt gövdesinin içeriği arasında bir eşleşme durumunda, tarayıcı uyarıyı yükseltir ve hemen geri döner. Herhangi bir eşleşme yoksa, tarayıcı, sunucuya ürünü yazmasını söyleyen birçok ASP dizesiyle istekler oluşturur. iki adet rastgele oluşturulmuş tamsayı. Yanıt gövdesi ürüne uyuyorsa, bir uyarı oluşturulur.

Command Injection

This rule submits *NIX and Windows OS commands as URL parameter values to determine whether or not the web application is passing unchecked user input directly to the underlying OS. The injection strings consist of meta-characters that may be interpreted by the OS as join commands along with a payload that should generate output in the response if the application is vulnerable. If the content of a response body matches the payload, the scanner raises an alert and returns immediately. Hata tabanlı eşleme denemelerinin hiçbirinin yapılmaması durumunda yanıtta çıktıyı döndür tarayıcı uyku talimatlarını yük olarak göndererek ve isteği gönderme arasındaki geçen süreyi karşılaştırarak kör bir enjeksiyon saldırısı yapmaya çalışır ve sezgisel gecikme alt sınırına karşı cevap alır. If the elapsed time is greater than this limit, an alert is raised with medium confidence and the scanner returns immediately.
Post 2.5.0 you can change the length of time used for the blind injection attack by changing the rules.common.sleep parameter via the Options 'Rule configuration' panel.

Client Browser Cache

This rule checks the headers of secure pages and reports an alert if they allow a browser to cache the page.

Siteler Arası Komut Dosyası Çalışması (yansıtılan)

Bu kural, bir 'güvenli' değer göndererek ve tüm bu değer (varsa) yanıtta oluştuğu konumları analiz başlar.
Daha sonra, özellikle her bir örnekten birinin bulunduğu yerde hedeflenmiş bir dizi saldırıyı gerçekleştirir., etiket niteliklerini, url niteliklerini, src niteliklerini, html yorumlarını destekleyen etiket niteliklerini içerir.

Siteler Arası Senaryo (kalıcı)

Bu kural benzersiz bir 'güvenli' değer göndererek başlar ve daha sonra örümcek, tüm uygulamada değerilerin bulunduğu tüm konumları bulmak için arar
Daha sonra 'yansıtılan' sürümün yaptığı gibi bir dizi saldırı gerçekleştirir ancak bu durumda diğer sayfalardaki hedef konumların hepsini kontrol eder.

CRLF Enjeksiyon

Bu kural, sunucuya bir parametre olarak bir "set Cookie" üstbilgisi önceki çeşitli CRLF özel karakterler gönderir. Eğer yanıt aynı set-cookie Başlığı içeriyorsa, bir uyarı oluşur ve tarayıcı hemen geri döner. Bu kural, Apache, IIS ile kullanılan desenler için yanıt gövdesini inceleyerek bir isteğin dizin listesine erişim sağlayıp sağlamadığını kontrol eder. ve diğer web sunucusu yazılımları.

External Redirect

This rule submits a variety of URL redirect strings as parameter values in a request, then examines the headers and bodies of responses to determine whether or not a redirect occurred and of what type. The cause of redirect is searched for in the "Location" and "Refresh" header fields, as well as by HTML meta tags and Javascript in the body of the response. An alert is raised including the redirection type and the scanner returns immediately.

Format String Error

Looks for indicators of format string handling errors in compiled code. Bu, düzenlenmiş karakterlere dayanan girdi metni dizelerini yerleştirerek yapar C kodunun biçimlendirilmiş çıktı üretmesini, kod çökmesini ve anormal oturum kapatmalarını öngörür.

Parameter Tampering

This rule submits requests with parameter values known to cause errors to be displayed to the user if handled improperly. Responses are checked to make sure that they return a server error status code, then compared with a normal HTTP response to make sure it does not raise an alert if the bad parameter has no effect on output. Finally, the content of the reponse body is compared against various patterns that may be found in Java servlet, Microsoft VBScript, OLE DB, JET, PHP and Tomcat errors. If a match is found, an alert is raised and the scanner returns immediately.

Path Traversal

This rule attempts to access files and directories outside of the web document root by constructing various combinations of pathname prefixes and local file targets for Windows and *NIX systems as well as Java servlets. If the body of the response matches a pattern corresponding to the current target file an alert is raised and the scanner returns immediately. If none of the common local file targets succeed, path traversal is attempted using the filename in the URL. As long as submitting an arbitrary filename does not return an OK status code but the real filename does, an alert is raised and the scanner returns immediately.

Uzak Dosya Dahil Et

Bu kural, parametre değerleri olarak harici URL'ler içeren bir dizi istek gönderir ve yanıt gövdesi ile sayfanın başlığı arasında bir eşleşme arar. Bu URL'lerde barındırılan. If there is a match between the expected string and the response body, and the header returned a status code of 200, an alert is raised and the scanner returns immediately.

Sunucu Tarafı Dahil Et

Bu kural sunucunun hangi işletim sistemi üzerinde çalıştığını kontrol eder, daha sonra karşılık gelen HTML SSI yönergesini parametre değeri olarak istekleri gönderir. Eğer yanıt gövdesi eşleşirse SSI başarılı olduğunu gösteren bir desen, bir uyarı oluşturulur ve tarayıcı hemen döner.

SQL Enjeksiyon

Bu tarayıcı, hata tabanlı, boolean tabanlı, Union tabanlı ve yığılmış sorgu SQL Enjeksiyon teknikleri kullanılarak, url parametrelerine ve URL'lerin geçerli ve geçersiz SQL sözdizimi parçaları ile saldırıya uğramasına bağlı olarak RDBMS'den bağımsız, SQL Enjeksiyon güvenlik açıklarını tarar.
This scanner may be able to fingerprint the RDBMS if the application throws a known RDBMS specific SQL error message.
This scanner does not exploit any RDBMS specific techniques, and so is the best SQL injection scanner to use as a starting point.
