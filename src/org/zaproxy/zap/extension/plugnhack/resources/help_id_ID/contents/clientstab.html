Tab Plug-n-Hack Klien

Plug-n-Hack mengizinkan Anda memantau acara klien (browser) untuk membantu menguji aplikasi HTML5.
Agar bisa mendukung berbagai macam browser modern semaksimal mungkin ini dilakukan dengan cara menyuntikkan javascript ke respon kembali ke browser.
Karena itu berarti mengubah respons fungsi ini harus diaktifkan secara manual.

Untuk menerima acara klien:

  • Akses halaman yang ingin anda pantau melakukan proxy melalui ZAP
  • Klik kanan di bagian node(s) yang relevan di Situs pohon dan pilih 'Monitor klien -> Termasuk sub'
  • Paksa browser anda untuk memuat ulang halaman
  • Peristiwa klien (seperti postMessage, klik, mouseover dll) kemudian akan tercantum di tab ini - pilih mereka untuk melihat rincian lengkap.

Javascript yang disuntikkan mengirimkan detak jantung ke ZAP, jadi ZAP dapat menampilkan semua klien 'aktif' di tab ini.
Dalam kasus ini klien 'aktif' adalah halaman yang saat ini terbuka di tab browser.
Anda dapat mengedit dan mengirim ulang acara apapun melalui item menu klik kanan asalkan klien asli aktif.

Anda juga bisa mencegat dan mengubah postMessages dengan cepat.
Pesan klien dicegat jika tombol 'istirahat pada semua permintaan' dipilih pada toolbar tingkat atas.
Anda juga dapat mengatur breakpoints klien kustom melalui sebuah tombol pada tab Clients.

Anda bisa membersihkan postMessages dengan cara yang sama seperti pesan lainnya - sorot string yang ingin Anda ubah di tab Permintaan, klik kanan dan pilih "Fuzz".
Pilihan ini hanya akan tersedia jika browser yang bersangkutan masih memiliki halaman yang terbuka, seperti PnH mengirimkan payloads ke browser.
Payloads 'XSS' standar dapat digunakan untuk mendeteksi kerentanan DOM XSS, namun Anda perlu memantau UI secara manual untuk melihat apakah ada yang berhasil.

PnH juga menambahkan file Custom fuzzing yang disebut "PlugnHack DOM XSS Oracle".
File ini didasarkan pada cheatsheet OWASP XSS dan meminta metode JavaScript 'xss' yang disuntikkan ke dalam halaman.
Metode ini memanggil API ZAP yang pada gilirannya menandai serangan yang relevan di UI - ini dikonfirmasi kerentanan DOM XSS, Meskipun (seperti biasa), Anda harus memeriksa apakah benar-benar dapat dieksploitasi dalam aplikasi yang Anda uji tanpa alat seperti ZAP.
