Active Scan Rules - Beta

Berikut beta kualitas active scan aturan termasuk dalam hal ini add-on:

Backup File Disclosure

Scan untuk umum bernama salinan cadangan dari file pada web server, yang dapat mengungkapkan informasi sensitif.

Cross-Domain Misconfiguration

Memeriksa apakah web server dikonfigurasi untuk memungkinkan akses Domain Lintas, dari yang berbahaya layanan pihak ketiga, misalnya. Saat ini pemeriksaan untuk wildcard di Adobe crossdomain.xml, dan di SilverLight ini clientaccesspolicy.xml.

Token CSRF

Memindai keberadaan token anti-CSRF.
Peringatan atas permintaan yang tampaknya tidak mengandung token anti-CSRF.
Pos 2.5.0 kamu dapat menentukan daftar pengenal yang dipisahkan dengan tanda koma di parameter rules.csrf.ignorelist melalui panel 'konfigurasi aturan' pilihan. FORM mana pun dengan nama atau ID yang cocok dengan salah satu pengidentifikasi ini akan diabaikan saat memindai bukti kontra-CSRF yang hilang. Hanya gunakan fitur ini untuk mengabaikan FORMs yang anda tahu adalah aman, misalnya bentuk-bentuk pencarian.

Injeksi Ekspresi Bahasa

Periksa jika web aplikasi adalah subjek untuk penyerangan injeksi Ekspresi Bahasa (EL), yang terjadi ketika sebuah aplikasi gagal untuk menetralkan secara cukup elemen-elemen khusus yang dapat memodifikasi pernyataan EL dimaksudkan sebelum di jalankan.

Kerentanan membukaSSL

Mendeteksi apakah server web rentan terhadap kerentanan membukaSSL yang terkejut, dengan memanfaatkannya. Untuk keterangan lebih lanjut lihat CVE-2014-0160.

Parameter polusi HTTP (HPP)

Menyediakan duplikat atau banyak parameter HTTP dengan nama yang sama dapat menyebabkan aplikasi atau situs web untuk menafsirkan nilai dengan cara yang tidak disengaja. Dengan memanfaatkan efek ini, berbahaya individu mungkin bisa bypass validasi input, memicu kesalahan atau memodifikasi internal nilai-nilai variabel. Ada perbedaan perlakuan dari duplikat parameter berdampak baik klien (browser) dan server.

Metode HTTP tidak aman

Mendeteksi (dan eksploitasi, tergantung pada pengaturan pindaian) metode HTTP yang dikenal tidak aman yang diaktifkan untuk URL.

Integer Overflow Error

Looks for indicators of integer overflows in compiled code that causes the web server to crash. Itu tidak ini dengan menempatkan beberapa string bilangan bulat yang dirancang untuk mencoba dan merangsang buruk tanggapan.

Padding Oracle

Scanner ini upaya untuk memanipulasi padding dienkripsi string untuk memicu respon kesalahan menunjukkan kemungkinan padding oracle kerentanan. Kerentanan tersebut dapat mempengaruhi setiap aplikasi atau framework yang menggunakan enkripsi tidak benar, seperti beberapa versi ASP.net Java Server Faces, dan Mono.

Remote Code Execution - CVE-2012-1823

Detect CVE-2012-1823 to perform Remote Code Execution on a PHP-CGI based web server.

Session Fixation

Session Fixation may be possible. Jika masalah ini terjadi dengan masuk URL (di mana pengguna mengotentikasi diri mereka sendiri untuk aplikasi), maka URL yang dapat diberikan oleh seorang penyerang, selain tetap session id, untuk korban, untuk kemudian mengasumsikan identitas korban menggunakan diberikan session id. Jika terjadi masalah dengan non-masuk halaman, URL dan tetap session id hanya dapat digunakan oleh penyerang untuk melacak aplikasi yang tidak berkepentingan tindakan pengguna. Jika terjadi kerentanan pada cookie bidang atau bentuk lapangan (POST parameter) bukan pada URL (GET) parameter, maka beberapa kerentanan juga mungkin diperlukan dalam rangka untuk mengatur cookie lapangan pada browser korban, untuk memungkinkan kerentanan yang dapat dimanfaatkan.

ShellShock - CVE-2014-6271

This scanner perform 2 attacks to detect servers vulnerable to CVE-2014-6271 aka ShellShock.
The first is a simple reflected attack and the second is a time based attack.

Post 2.5.0 you can change the length of time used for the attack by changing the rules.common.sleep parameter via the Options 'Rule configuration' panel.

Pengungkapan sumber kode - CVE-2012-1823

Eksploitasi CVE-2012-1823 untuk mengungkapkan kode sumber PHP sisi server pada server web berbasis PHP-CGI.
Hanya analisis tanggapan yang berbasis teks (HTML, JS, JSON, XML, dll), untuk menghindari kesalahan positif yang mungkin terjadi pada gambar atau konten biner lainnya.

Pengungkapan sumber kode - SVN

Menggunakan metadata repositori kode sumber subversi untuk memindai berkas yang berisi kode sumber pada server web.

Pengungkapan sumber kode - /WEB-INF

Mengeksploitasi kehadiran folder / WEB-INF yang tidak dilindungi untuk mendownload dan mendekompilasi kelas Java, untuk mengungkapkan kode sumber Java.

SQL injeksi - Hipersonik (berbasis waktu)

Pemindai ini menggunakan sintaks SQL spesifik Hipersonik untuk mencoba menginduksi penundaan waktu dalam pernyataan SQL yang disebut oleh halaman.
Jika kueri yang tidak dimodifikasi tidak terpengaruh oleh penundaan waktu, dan penundaan kueri yang dimodifikasi dapat dikendalikan, ini adalah indikasi kerentanan SQL injeksi berbasis waktu dalam database SQL hipersonik.
Pemindai ini sensitif terhadap waktu, dan seharusnya hanya digunakan untuk menemukan SQL yang keras kepala dan tidak jelas kerentanan injeksi dalam database hipersonik yang dicurigai.
Untuk alasan ini, jumlah benang pemindai aktif harus diatur seminimal mungkin saat menggunakan pemindai ini, untuk meminimalkan beban pada server web, server aplikasi, dan database, untuk menghindari kesalahan positif yang disebabkan oleh penundaan beban bukan dengan penundaan injeksi SQL.
Tes pemindaian hanya untuk waktu dasar kerentanan injeksi SQL

Pos 2.5.0 Anda dapat mengubah jangka waktu yang digunakan untuk menyerang dengan mengubah rules.common.sleep parameter melalui pilihan panel 'konfigurasi pengaturan'.

Pemindai ini menggunakan MySQL-spesifik SQL syntax untuk mencoba untuk menginduksi penundaan waktu dalam pernyataan SQL yang disebut oleh halaman.
Jika pertanyaan yang tidak dimodifikasi tidak terpengaruh oleh penundaan waktu, dan penundaan pertanyaan yang dimodifikasi dapat dikendalikan, Ini adalah indikasi kerentanan SQL Injeksi berbasis waktu dalam database MySQL.
Pemindai ini sensitif terhadap waktu, dan seharusnya hanya digunakan untuk menemukan SQL yang keras kepala dan tidak jelas kerentanan injeksi dalam database MySQL yang dicurigai.
Untuk alasan ini, jumlah rangkaian pemindai aktif harus diatur seminimal mungkin saat menggunakan pemindai ini, untuk meminimalkan beban pada server web, server aplikasi, dan database, untuk menghindari kesalahan positif yang disebabkan oleh penundaan beban bukan dengan penundaan injeksi SQL.
The scanner tests only for time-based SQL injection vulnerabilities.

Penghitungan nama pengguna

Ada kemungkinan untuk menghitung nama pengguna, berdasarkan tanggapan HTTP yang berbeda saat nama pengguna yang sah dan tidak sah disediakan. Ini akan sangat meningkatkan kemungkinan keberhasilan password brute-forcing serangan terhadap sistem. Catatan yang positif palsu kadang-kadang dapat diminimalisir dengan meningkatkan 'Attack Strength' Pilihan di ZAP. Silakan periksa manual 'Other Info' lapangan untuk mengkonfirmasi jika hal ini benar-benar masalah. Scanner ini dilewati jika tidak ada konteks yang didefinisikan yang menggunakan Otentikasi berbasis Bentuk, dan hanya berjalan melawan URL diidentifikasi sebagai masuk URL dari konteks.

Injeksi XPath

Seperti yang dijelaskan oleh OWASP: "Serangan injeksi XPath terjadi saat situs web menggunakan informasi yang dipasok pengguna untuk kueri XPath untuk data XML. Dengan mengirimkan sengaja salah bentuk informasi ke situs web, seorang penyerang dapat mengetahui bagaimana data XML terstruktur, atau akses data yang dia dapat biasanya tidak memiliki akses untuk. Dia bahkan mungkin dapat meningkatkan hak-nya di situs web jika data XML yang digunakan untuk otentikasi (seperti XML pengguna berbasis file) atau otorisasi." Scanner ini mencoba untuk mengidentifikasi seperti kelemahan-kelemahan.

XXE

This component attempts to identify applications which are subject to XML eXternal Entity (XXE) attacks. Applications which parse XML input may be subject to XXE when weakly or poorly configured parsers handle XML input containing reference to an external entity such as a local file, HTTP requests to internal or tertiary systems, etc.

It requires the Callback extension, so will not work if this extension is disabled or removed. Hal ini juga dianjurkan bahwa anda menguji bahwa Callback ekstensi adalah dikonfigurasi dengan benar untuk situs target. Jika sistem target tidak dapat terhubung ke Callback Alamat kemudian beberapa XXE kerentanan tidak akan terdeteksi.
